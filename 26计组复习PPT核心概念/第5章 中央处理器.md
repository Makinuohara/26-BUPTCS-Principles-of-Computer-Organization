# 第5章 中央处理器

**核心概念**

CPU 功能包括指令控制、操作控制、时间控制、数据加工和中断处理。CPU 通常由运算器和控制器组成，现代结构还常把 Cache 作为 CPU 内部重要部分讨论。

常用寄存器：PC 保存下一条指令地址；IR 保存当前指令；MAR/AR 保存访存地址；MDR/DR 保存访存数据；通用寄存器保存操作数和结果；PSW 保存状态标志。

指令周期是取出并执行一条指令的总时间，通常包括取指、间址、执行、中断等周期。CPU 周期/机器周期由若干时钟周期组成。

数据通路是数据流经的路径，包括寄存器、ALU、MUX、总线等。控制信号决定数据从哪里来、到哪里去、何时写入。

微程序控制：一条机器指令由一段微程序解释执行；微程序由微指令组成；微指令发出微命令；微命令控制微操作。

硬布线控制器用组合逻辑直接产生控制信号，速度快但修改困难。微程序控制易扩展、易修改，但速度较慢。

**易错辨析**

PC 不是当前指令内容，IR 不是下一条指令地址。MAR 和 MDR 不要混，前者是地址，后者是数据。

指令周期不等于取指周期，也不等于执行周期。

微命令是控制信号，微操作是执行动作。一条机器指令不是一条微指令。

水平型微指令字长长、并行性强、速度快；垂直型微指令字长短、并行弱、速度慢。

流水线提高吞吐率，不缩短单条指令延迟。冒险会降低实际效率。

**典型时序/控制题步骤**

取指公共微操作：

```text
PC -> AR
Read
DR -> IR
PC + 1 -> PC
指令译码
```

执行阶段按指令功能倒推数据路径。若是 ADD R2,(R1) 这类题，先由 R1 指向的存储单元取数，再与 R2 相加并写回。若是 JMP R1，按题意把 R1 与 PC 相加或把目标地址送 PC。

写控制信号时按数据通路图列输出门、输入门、ALU 控制、存储器读写信号。单总线同一时刻只能有一个输出源。

流水线公式：k 级流水线执行 n 个任务，流水时间 Tk=(k+n-1)τ，理想非流水时间 T0=nkτ，加速比 S=nk/(k+n-1)。

**结合官方样题推测的考法**

官方样题第七大题给模型机数据通路，要求写 ADD 和 JMP 的执行阶段微操作和有效控制信号。第 5 章必须会看图写数据流和控制信号。

选择/判断还会考 PC/IR、指令周期、机器指令与微指令关系、RISC 是否微程序控制、硬布线与微程序比较。

**补充题可能延伸**

10 套整理可能考水平/垂直微指令、控制信号逻辑表达式、流水线加速比、RAW/WAR/WAW 冒险判断。

流水线冒险：结构冒险是争资源；数据冒险有 RAW、WAR、WAW；控制冒险由分支引起。解决方法包括停顿、转发、重排、分支预测。

**最后一周背诵清单**

- PC：下一条指令地址；IR：当前指令。
- MAR 地址，MDR 数据。
- 指令周期 -> CPU 周期 -> 时钟周期。
- 取指微操作模板。
- 机器指令 -> 微程序 -> 微指令 -> 微命令/微操作。
- 硬布线快难改，微程序慢易改。
- 数据通路题先看图，再写取指，再写执行，再配控制信号。
