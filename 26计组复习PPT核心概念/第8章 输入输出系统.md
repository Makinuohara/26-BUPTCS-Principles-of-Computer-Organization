# 第8章 输入输出系统

**核心概念**

输入输出系统由 I/O 设备、I/O 接口、设备控制器、I/O 管理软件组成。I/O 接口用于解决 CPU 与外设在速度、格式、时序和控制方式上的差异，通常包含数据寄存器、状态寄存器、命令寄存器、地址译码和控制逻辑。

I/O 编址分独立编址和统一编址。独立编址下 I/O 端口和主存地址分开，常用 IN/OUT 指令；统一编址又称存储器映像 I/O，把接口寄存器放入主存地址空间，可用访存指令访问，但占用地址空间。

四种 I/O 控制方式按 CPU 参与程度从高到低为：程序查询、程序中断、DMA、通道。

**程序查询方式**

CPU 启动外设后反复读状态寄存器，Ready 未就绪则循环等待，就绪后传送数据。优点是硬件简单，缺点是 CPU 忙等待，效率最低。

**程序中断方式**

外设准备好后通过接口向 CPU 发中断请求。CPU 在一条指令结束后响应，保存断点，识别中断源，转入中断服务程序，服务完恢复现场并返回。

中断提高 CPU 与外设并行性，但数据传送仍通常由 CPU 执行输入/输出指令完成。

**DMA 方式**

DMA 是直接存储器访问，适合主存和 I/O 设备之间高速成批传送。CPU 先初始化 DMA 控制器，传送阶段由 DMAC 接管总线，直接控制主存与外设交换数据，结束后用中断或状态位通知 CPU。

DMA 方式包括成组传送、周期窃取、透明 DMA。成组传送速度高但 CPU 停顿明显；周期窃取每次占用一个存储周期；透明 DMA 交替访存，CPU 基本不停顿。

**通道和外围处理机**

通道是代替 CPU 管理外设的专用处理器，能执行有限 I/O 指令。结构常为主机、通道、设备控制器、I/O 设备。选择通道适合高速设备独占块传送；字节多路通道适合低中速字符设备；数组多路通道按数据块交叉使用，效率高但复杂。

**易错辨析**

程序查询和程序中断方式下，外设与内存之间的数据通常通过 CPU 交换；DMA 和通道方式下，数据主要在外设与主存之间直接交换。

DMA 期间 CPU 不是绝对不能执行任何指令。成组传送时 CPU 可能长时间失去总线；周期窃取只短暂停顿；透明 DMA 下 CPU 可继续运行。官方样题这种绝对说法应判错。

中断方式不等于数据直接进内存，中断只是通知 CPU。

**典型流程题/比较题**

查询流程：启动外设、读状态、判断 Ready、传送数据、循环到完成。

中断流程：请求、判优、响应、保存断点、关中断、识别中断源、取入口、保护现场、服务、恢复现场、返回。

DMA 流程：CPU 初始化 DMAC、外设请求、DMAC 请求总线、CPU 让出总线、DMAC 传送、地址加 1、字计数减 1、结束通知 CPU。

中断与 DMA 异同：相同点是都减少 CPU 忙等待、都可提高并行性、结束时可中断通知 CPU。不同点是中断传送数据仍由 CPU 完成，DMA 由 DMAC 控制主存与外设直接传送，适合高速批量数据。

**结合官方样题推测的考法**

官方样题第 8 章重点是判断和简答：程序查询/中断方式数据是否经过 CPU，DMA 期间 CPU 是否一定停顿，程序中断方式和 DMA 方式的异同。

**补充题可能延伸**

可能考多级中断优先级、IM/EI 屏蔽、I/O 编址区别、DMA 请求服务时间图、三类通道比较。

**最后一周背诵清单**

- 查询：CPU 主动轮询，效率低。
- 中断：外设通知 CPU，CPU 服务，数据仍经 CPU。
- DMA：DMAC 接管总线，主存和外设直接传送。
- DMA 三方式：成组、周期窃取、透明。
- 通道：专用 I/O 处理器。
- 官方重点：数据是否经过 CPU，CPU 是否一定停顿，中断和 DMA 异同。
