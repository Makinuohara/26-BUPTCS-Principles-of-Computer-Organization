# 第1章 计算机概述

**核心概念**

计算机系统由硬件和软件组成。软件通常分为系统软件和应用软件，系统软件包括操作系统、编译器、汇编器、解释程序、数据库管理系统等。

冯诺依曼结构是本章主线：计算机由运算器、控制器、存储器、输入设备、输出设备组成；程序和数据同存在存储器中；机器内部用二进制表示；指令由操作码和地址码组成；计算机按存储程序方式自动、连续地取指执行。

硬件能直接执行的是机器语言。高级语言、汇编语言都要经过编译、解释或汇编后才能运行。

计算机层次可按从低到高理解为：微程序或硬布线级、一般机器级、操作系统级、汇编语言级、高级语言级。

**容易考的定义/辨析**

机器字长是 CPU 一次能处理的二进制数据位数，通常与寄存器位数、运算器位数有关。机器字长影响数的表示范围、精度和硬件成本。

处理机字长是处理机运算器一次能完成二进制运算的位数。

存储程序的核心意义是保证计算机自动、连续地执行程序，不是简单地节省编程时间。

MIPS 是每秒百万条指令，CPI 是每条指令平均所需时钟周期数。两者都属于性能指标，但 MIPS 会受指令系统影响，不能绝对比较所有机器。

**公式或单位换算**

1B = 8bit。1KB = 2^10B，1MB = 2^20B，1GB = 2^30B。

时钟周期 T = 1 / 主频 f。

CPU 执行时间 = 指令条数 × CPI × 时钟周期。

平均 CPI = 各类指令比例 × 对应 CPI 的总和。

MIPS = 主频 / CPI / 10^6。

**结合官方样题推测的考法**

官方样题已出现：软件分为系统软件和应用软件；存储程序的重要意义；硬件直接执行机器语言；机器字长错误说法；MIPS、机器字长等术语解释。

选择题常问概念辨析，简答题可能要求解释机器字长、相联存储器、微程序、MIPS。性能题可能把更快处理器、优化代码、加入 Cache、多处理器放在一起问是否能缩短响应时间。

**补充题可能延伸**

10 套整理常考冯诺依曼机：绝大多数计算机从系统结构看仍属于冯诺依曼计算机，其基本工作方式是按地址访问并顺序执行指令。

可能问 CPU 基本组成。传统回答是运算器和控制器；现代语境可补充 Cache，但不要把主存写入 CPU。

**最后一周背诵清单**

- 计算机系统 = 硬件 + 软件。
- 软件 = 系统软件 + 应用软件。
- 冯诺依曼五大部件：运算器、控制器、存储器、输入设备、输出设备。
- 存储程序：程序和数据同存，按地址访问，自动连续执行。
- 机器字长：CPU 一次处理的数据位数。
- CPU 执行时间公式和 MIPS 公式。
