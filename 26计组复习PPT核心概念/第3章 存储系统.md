# 第3章 存储系统

**核心概念**

存储系统追求容量大、速度快、成本低，但单一存储器很难同时满足，因此采用层次结构：寄存器、Cache、主存、辅存。越靠近 CPU，速度越快、容量越小、成本越高。

Cache-主存层次解决 CPU 与主存速度不匹配；主存-辅存层次解决主存容量不足。基础是时间局部性和空间局部性。

SRAM 速度快、不刷新、成本高、集成度低，常作 Cache。DRAM 集成度高、容量大、成本低，但速度慢、需刷新，常作主存。

主存扩展有位扩展、字扩展、字位同时扩展。位扩展扩字长，地址线和控制线共用，数据线分开；字扩展扩字数，高位地址译码产生片选。

Cache 映射方式：全相联、直接映射、组相联。替换算法有随机、FIFO、LFU、LRU。写策略有写穿透、写回、写分配、非写分配。

虚拟存储器给用户更大逻辑地址空间。虚拟地址经页表/TLB 转换为物理地址。页内地址转换前后不变。

**易错辨析**

Cache 的目的不是扩大主存容量，而是加快访存速度。扩大地址空间主要靠虚拟存储器。

Cache 与主存映射由硬件自动完成，不由程序员、操作系统或编译器在程序执行时直接完成。

CPU 可直接访问 Cache 和主存，不能直接访问辅存。辅存数据必须先调入主存。

DRAM 集成度高但速度不比 SRAM 快。判断题中“DRAM 集成度更高、速度更快”是错的。

虚拟地址是否在主存，先看页表有效位；有效位为 0 表示缺页，不必继续判断 Cache。

磁盘访问时间不能只算传输时间，还要加平均寻道时间和平均旋转等待时间。

**典型计算步骤**

芯片数量：芯片数 = 目标总容量 / 单片容量。若位宽不够先位扩展，若字数不够做字扩展。

官方样题主存设计：16 条地址线表示 64KB 地址空间。最高 4KB 为 F000H-FFFFH，紧邻 4KB 为 E000H-EFFFH，最低 16KB 为 0000H-3FFFH。低位地址接片内地址，高位地址经 74138 译码片选，RAM 接 R/W，ROM 只读。

Cache 地址字段：块大小 2^b 字节则块内地址 b 位。直接映射字段为标记、行号、块内地址；组相联为标记、组号、块内地址；全相联为标记、块内地址。

直接映射行号 = 主存块号 mod Cache 行数。组相联组号 = 主存块号 mod 组数。

虚拟地址与 Cache 联合题：先拆虚拟地址，查页表形成物理地址，再拆物理地址查 Cache。官方样题中 16MB 虚拟地址为 24 位，1MB 主存物理地址为 20 位，4KB 页页内地址为 12 位，8 行直接映射行号 3 位，32B 块内地址 5 位，标记 12 位。

磁盘平均访问时间 = 平均寻道时间 + 平均旋转等待时间 + 数据传输时间。7200RPM 一圈 8.33ms，平均等待 4.17ms；512B、1MB/s 传输约 0.512ms；加 20ms 约 24.7ms。

**结合官方样题推测的考法**

本章在官方样题中权重很高。小题考 Cache 目的、Cache 映射由谁完成、虚拟存储器作用、SRAM/DRAM、CPU 是否能直接访问辅存。大题考主存芯片扩展与连接图、虚拟存储器和 Cache 联合判断。

**补充题可能延伸**

10 套整理常考 EEPROM、DRAM 地址线/数据线数、双端口存储器、多模块交叉存储器、Cache 命中率、平均访问时间、三级存储平均访问时间、页面替换算法。

Cache 命中率 h = 命中次数 / 总访问次数。平均访问时间常用 ta = htc + (1-h)tm；现代 AMAT 写成命中时间 + 缺失率 × 缺失损失。

**最后一周背诵清单**

- Cache 解决速度问题，虚存解决容量/地址空间问题。
- SRAM 快、贵、不刷新；DRAM 慢、便宜、需刷新。
- 位扩展扩位数，字扩展扩容量。
- 直接映射：标记、行号、块内地址。
- 页式虚存：虚页号 + 页内地址；页内地址不变。
- 官方联合题字段：24 位虚拟地址、20 位物理地址、12 位页内、3 位行号、5 位块内、12 位标记。
- 磁盘平均时间：寻道 + 半圈等待 + 传输。
